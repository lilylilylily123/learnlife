# NFC Attender — Production Hardware v1

Build guide for the desk-unit version of the LearnLife NFC Attender. Turns the
working bench rig (ESP32 + PN532 + OLED + buzzer on two breadboards) into a
single 3D-printed enclosure with a soldered perfboard, panel-mount power
switch, and reset button.

**Status:** v1 design; one unit for the Hub front desk.

## Wiring schematic

See `schematic.txt`. Summary:

- USB-C 5V → SPDT switch → ESP32 VIN
- Shared 3.3V/GND rails to PN532 + OLED
- I2C: SDA=GPIO 21, SCL=GPIO 22 (PN532 + OLED on shared bus)
- Buzzer: GPIO 25
- Reset button across ESP32 EN ↔ GND
- PN532 DIP switches: 1=ON, 2=OFF (I2C mode, address `0x24`)

No level shifters needed. No firmware changes from the bench config.

## Perfboard layout

See `perfboard-layout.txt`. All modules (ESP32, PN532, OLED) live in female
sockets so they can be popped out for re-flashing or replacement without
desoldering.

## 3D-printed enclosure

Two parts in `enclosure/`:

- `enclosure.scad` — parametric OpenSCAD source. Top-of-file parameters
  control every dimension (footprint, wall thickness, component cutouts,
  fastener sizes).
- `stl/bottom_tray.stl` and `stl/top_lid.stl` — generated by `render.sh`.
  Not checked in until verified (run `./render.sh` after installing
  OpenSCAD).
- `render.sh` — regenerates STLs after editing the .scad.

**Form:** rectangular project box, 100mm wide × 85mm deep × 36mm tall (30mm
tray + 6mm lid). Flat top.

Why rectangular and not the wedge originally proposed: the wedge geometry
put the PN532 antenna ~40mm from the angled top surface (past the reliable
read range of ~30mm) when everything lived on a single perfboard. Fixing
this would require mounting the PN532 to the lid with flying wires, doubling
the assembly complexity for v1. A low rectangular box keeps every component
on the perfboard with the PN532 antenna a consistent ~10–15mm from the lid.
A wedged v2 is straightforward later — split PN532/OLED to lid-mounted
sub-board.

The top lid is thinned to 1.6mm directly above the PN532 antenna (a 45mm
square pocket on the underside) so card reads come through at full
sensitivity. The OLED window is cut all the way through the lid.

Print settings: see `enclosure/print-settings.md`.

Installing OpenSCAD on macOS:
```bash
brew install --cask openscad
# First run is blocked by Gatekeeper. Either:
#   - Right-click /Applications/OpenSCAD-2021.01.app → Open → Open
#   - or System Settings → Privacy & Security → Allow Anyway
```

## Build order

### Phase A — Soldering (allow 2–3 hours for a first solder)

1. **Dry-fit components** on the perfboard against the layout diagram.
   Confirm the ESP32 USB-C lines up with where the back-wall cutout will be.
2. **Solder female header sockets first** (warm-up joints): ESP32 (2× 19-pin),
   PN532 (4-pin), OLED (4-pin).
3. **Solder power & ground rails** along the long edges using bare 24 AWG.
   Solder every other column for mechanical stability.
4. **Wire I2C and buzzer**: SDA/SCL between ESP32 GPIO 21/22 and the PN532/OLED
   sockets; buzzer GPIO 25 to its terminal pads.
5. **Attach flying leads** for the power switch (USB-C 5V → switch → ESP32 VIN,
   ~80mm) and reset button (ESP32 EN ↔ GND, ~80mm).
6. **Bench test** the soldered board before printing the case. Plug the ESP32
   into USB-C, flip the toggle, confirm boot logs, tap a card, confirm
   OLED + buzzer + PocketBase write. Do not proceed if anything regresses.

### Phase B — Printing (allow 6–8h print time)

1. Install OpenSCAD (`brew install --cask openscad`) and render the STLs:
   `cd enclosure && ./render.sh`.
2. **Print `bottom_tray.stl` in PLA first** for a fit check (~3h on a Bambu
   P1S). Drop the perfboard in — check standoff alignment and side-cutout
   positions. Adjust parameters in `enclosure.scad` and reprint if needed.
3. **Print `top_lid.stl` in PETG** (~3h). PETG handles desk wear better than
   PLA. Print the angled face on the build plate (lay the lid with the flat
   underside down) for the cleanest tap surface.
4. **Install M3 heat-set inserts** into the four bosses on the bottom tray
   rim. Use a soldering iron at ~200°C; press straight down until flush.

### Phase C — Assembly

1. Mount the perfboard to the bottom tray with 4× M2.5×6mm self-tap screws
   into the standoffs.
2. Glue the piezo disc to the inside of the top lid, centered over the
   buzzer grille holes. Hot glue or two-part epoxy.
3. Solder the buzzer's flying leads to the perfboard's GPIO 25 + GND pads
   (these are the only solder joints made during assembly).
4. Mount the SPDT switch through its right-side cutout; tighten the
   panel-mount nut.
5. Mount the tactile button through its right-side cutout; tighten its nut.
6. Route the USB-C cable (the ESP32's onboard USB-C points out through the
   back-wall cutout).
7. Drop the lid on, drive 4× M3×8mm socket-caps into the heat-set inserts.
8. Stick rubber feet into the recesses on the bottom face.

## Verification

Power on via USB-C with the switch ON. The unit should:

- Show boot splash on OLED, connect WiFi (serial `[boot]` logs at 115200 baud).
- Read a known card tapped on the labeled tap zone within ~5mm; display name +
  verdict; play matching buzzer cue; write attendance row to PocketBase.
- Reject unknown cards with the unknown-card cue.
- Power switch OFF → fully unpowered. Switch ON → clean boot.
- Reset button → ESP32 reboot without affecting USB host connection.

**Read distance check:** card should reliably trigger at 0–5mm above the tap
pad. If not:
- Lower `nfc_window` in `enclosure.scad` (default 1.6mm).
- Raise `pb_clear` (perfboard standoff height) to bring the PN532 closer to
  the lid.

**Soak test:** leave powered on overnight. Verify in the morning that it's
still responsive and the reset-reason in the serial log is still `power-on`
(no unexpected reboots).

## Out of scope (v1)

- Battery / charging.
- RGB LED (mount area marked on perfboard but unwired).
- DS3231 RTC (NTP is sufficient).
- Acrylic OLED cover (display is recessed).
- IP / dust sealing (dry indoor desk unit).
- Multi-unit reproducibility (no PCB).

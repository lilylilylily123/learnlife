# LearnLife

## Overview
LearnLife is an innovative platform designed to enhance learning experiences through interactive and engaging methods. Whether you are a student, educator, or lifelong learner, our platform provides tools and resources to help you achieve your educational goals.

## Installation/Setup Instructions
1. **Clone the repository**:
   ```bash
   git clone https://github.com/<owner>/learnlife.git
   cd learnlife
   ```
2. **Install the required dependencies**:
   ```bash
   npm install
   ```
3. **Run the application**:
   ```bash
   npm start
   ```

For more detailed instructions, refer to the documentation in the repository.

## Code Review Findings (ll-calendar)

The following findings were generated by a full automated review of the **ll-calendar** (Luminous Scholar) Expo + React Native app.

---

### Phase 1: Code Quality & Architecture

#### Critical

- **C-1. PocketBase Filter Injection** (`lib/pocketbase.ts:74`) ‚Äî `userId` is interpolated directly into a PocketBase filter string via template literal. This is equivalent to SQL injection for PocketBase queries. Use parameterized filters (`pb.filter()`).
- **C-2. AuthContext `isAuthenticated` Is Not Reactive** (`context/AuthContext.tsx:33`) ‚Äî `isAuthenticated` is computed as `pb.authStore.isValid` at render time but only re-renders when `user` state changes. Token expiry won't trigger a re-render, leaving the UI in a stale authenticated state.

#### High

- **H-1. `fetchCalendarEvents` Ignores Date Range Parameters** (`lib/pocketbase.ts:68-78`) ‚Äî `_monthStart` and `_monthEnd` are accepted but unused. All records are fetched with `getFullList`, creating unbounded data growth.
- **H-2. `register()` Uses `any` Type** (`lib/pocketbase.ts:36`) ‚Äî `userData: any` disables all type checking. Also silently calls `login()` as a side effect.
- **H-3. Monolithic `index.tsx` ‚Äî 825 Lines** (`app/(tabs)/index.tsx`) ‚Äî Login screen + dashboard + 484 lines of styles in one file. Violates SRP.
- **H-4. Hardcoded Colors Throughout** (all screen files) ‚Äî `#2D1B4E`, `#8A7E9E`, `#C4F34A`, `#F9FAFC` repeated dozens of times. `constants/theme.ts` exists but is only used by boilerplate.
- **H-5. `error: any` in All Catch Blocks** (5 locations) ‚Äî Bypasses TypeScript strict mode.
- **H-6. No Dark Mode in Custom Screens** (all custom screens) ‚Äî Light-mode colors hardcoded despite `ThemeProvider` and `useColorScheme` being set up.
- **H-7. `BottomNav` Uses `router.push`** (`components/bottom-nav.tsx:29`) ‚Äî Tab navigation via `push` builds up a deep back-stack. Should use `replace` or `navigate`.
- **H-8. Console Logging Leaks Data** (`app/(tabs)/calendar.tsx:58-65`) ‚Äî User IDs, event titles, and full records logged via `console.log` + `JSON.stringify` in production paths.

#### Medium

- **M-1. `useFocusEffect` Closure Captures Stale `loadEvents`** (`app/(tabs)/calendar.tsx:72-76`) ‚Äî `loadEvents` not wrapped in `useCallback`.
- **M-2. `FlatList` Nested Inside `ScrollView`** (`app/(tabs)/index.tsx:208-231`) ‚Äî Triggers RN warning; replace with horizontal `ScrollView` for small static data.
- **M-3. Dashboard Data Is Hardcoded** (`app/(tabs)/index.tsx:21-32, 179-195`) ‚Äî Static mock data (names, dates, events) presented as real.
- **M-4. `ThemedText` Style Uses Repetitive Ternaries** (`components/themed-text.tsx:24-28`) ‚Äî Use record lookup instead.
- **M-5. `expandEvents` Doesn't Check Start Date for Weekly Recurrence** (`lib/calendar-utils.ts:80-101`) ‚Äî Shows occurrences before the event's actual start date.
- **M-6. Unused Boilerplate Components** (~8 files) ‚Äî `HelloWave`, `ParallaxScrollView`, `ExternalLink`, `Collapsible`, `IconSymbol`, `explore.tsx`, `modal.tsx`.
- **M-7. `parallax-scroll-view.tsx` Has Unused Style** (line 67) ‚Äî `styles.container` defined but never used.
- **M-8. Inconsistent Import Path Style** ‚Äî Mixed `@/` alias and relative paths for the same modules.
- **M-9. `recurrence_end` Boundary Check Fragile** (`lib/calendar-utils.ts:96`) ‚Äî Works but relies on time-component assumptions.
- **M-10. No End-Time Validation in Create Event** (`app/(modals)/create-event.tsx:143-191`) ‚Äî User can create events where end < start.

#### Low

- **L-1. Copyright Year Hardcoded to 2024** (`app/(tabs)/index.tsx:170`)
- **L-2. `unstable_settings` May Be Unnecessary** (`app/_layout.tsx:13`)
- **L-3. `loadEvents` Not Debounced** ‚Äî Rapid month switching causes concurrent requests.
- **L-4. Notification Dot Hardcoded** (`components/bottom-nav.tsx:16`)
- **L-5. Unused `Modal` Import** (`app/(modals)/create-event.tsx:10`)
- **L-6. `Fonts` Constant Unused by App Screens** (`constants/theme.ts:30`)
